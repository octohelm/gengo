# gengo AGENTS

## 适用范围

- 作用域：仓库根目录及未被子模块 `AGENTS.md` 覆盖的路径
- 目标：约束全局变更边界、执行优先级与接管机制

## 可改范围与禁区

- 可改：与当前任务直接相关的仓库内文件
- 禁区：未授权路径、生产环境操作、破坏性历史改写（如 `reset --hard`、强推）

## 停止条件

- 需求边界不清，存在多种互斥实现
- 变更会影响现有生成行为但无法完成最小验证
- 发现规则冲突且无法在当前任务内消解

## 人工接管条件

- 需要仓库外系统权限或敏感配置
- 需要决定是否新增模块、生成器或新的执行入口
- 需要发布窗口、审批流或数据迁移计划

## 规则优先级

1. 仓库根 `AGENTS.md`（本文件）
2. 模块 `AGENTS.md`
3. 任务临时约束

## 新规则审校机制

1. 新规则必须附带：目标、边界、风险、回滚方式
2. 启用前必须执行冲突检查（全局/模块/任务三层）
3. 冲突未消解前不得生效

## 协同约束

1. `README.md` 负责仓库语义与导航，`justfile` 负责执行入口，不在本文件混写。
2. 涉及新生成指令、生成结果语义或公开 API 的变更，应优先保持向后兼容；无法兼容时先明确影响范围。
3. 新增子模块前必须先确认是否新增，以及采用单入口还是多入口。
4. `justfile` 优先聚合稳定的 Go 工具链入口，而不是扩散一次性 shell 命令。

## 全局编码倾向

1. 不做与目标无关的功能扩展、顺手重构或批量风格清理。
2. 优先通过参数传递上下文，避免可变全局状态。
3. 避免过度工程、过度防御、过早抽象与无必要注释。
4. 不自行安装依赖、工具或系统组件。
5. 缺依赖导致阻塞时，明确记录为环境前置条件。
6. 非极小任务先拆分再执行。
7. 测试描述优先准确中文，直接表达业务语义、输入条件与预期。
8. 无法补测时必须说明缺口与原因。
